ML PIPELINE

PROCESSING PIPELINE
01

DATA INGESTION

Load cyberfeddefender_dataset.csv. Inspect shape, dtypes, and missing values.

02

PREPROCESSING

Drop Timestamp, Source_IP, Destination_IP. Label-encode Protocol. Select numeric features.

03

EDA

Class distribution, feature histograms, boxplots by label, correlation heatmap.

04

SPLITTING & SCALING

Stratified 99/1 train-test split. StandardScaler for GNB/BNB/LR/RF, MinMaxScaler for MNB (non-negative required).

05

TRAINING

5 classifiers: Gaussian NB, Multinomial NB, Bernoulli NB, Logistic Regression, Random Forest (100 trees).

06

EVALUATION

Accuracy, Precision, Recall, F1, ROC-AUC. Confusion matrices per model.

TECHNOLOGY STACK

Python-based ML pipeline using industry-standard libraries:

Python 3.10 pandas NumPy scikit-learn matplotlib seaborn Google Colab GaussianNB MultinomialNB BernoulliNB LogisticRegression RandomForestClassifier StandardScaler MinMaxScaler LabelEncoder ROC-AUC
DATASET FEATURES

Key network traffic features used for classification:

Packet_Length Duration Bytes_Sent Bytes_Received Flow_Packets/s Flow_Bytes/s Avg_Packet_Size Protocol Header_Length TTL Label (Target) Attack_Type
BINARY CLASSIFICATION TASK

The model predicts a binary Label:

0 → NORMAL — Legitimate network traffic with expected behavior patterns.

1 → ATTACK — Malicious traffic including DDoS, port scans, botnet activity, and intrusions.
